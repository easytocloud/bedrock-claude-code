## P1 — High Priority

### ARCH-1: `panel.ts` is a God object 🔲
**File**: `src/panel.ts` (17KB)
Mixes 6 concerns: migration logic (lines 25–151), webview rendering (229–237), save/apply logic (286–343), directory picker UI (345–360), API calls (362–435), draft persistence (439–495).

**Fix**: Extract into dedicated modules:
- `src/migration.ts` — `migrateExistingSettings()` and detection helpers
- `src/draft.ts` — `DraftStore` class with `save()`, `load()`, `clear()`
- `src/modelFetch.ts` — `fetchLocalModels()`, `fetchBedrockModels()` with shared error handling

### ARCH-2: `ScopeAssignment` type is not discriminated 🔲
**File**: `src/types.ts:77-82`
`presetId` is optional on all modes but assumed present when `mode === 'preset'`. Runtime hazard invisible to the compiler.

**Fix**:
```typescript
export type ScopeAssignment =
  | { mode: 'preset'; presetId: string }
  | { mode: 'manual' }
  | { mode: 'inherit' };
```

### ARCH-3: No runtime schema validation on JSON load 🔲
**Files**: `src/profiles.ts:100-102`, `src/panel.ts:456`, `src/claudeSettings.ts:33-35`, `src/claudeJson.ts:26-28`, `src/mcpJson.ts:24-26`
All five files cast parsed JSON to typed interfaces without checking required fields. Corrupted files cause silent runtime failures.

**Fix**: Add `validateProfileStore(parsed: unknown): parsed is ProfileStore` type guard in `profiles.ts`. Apply same pattern to other readers.

### ARCH-4: Provider logic embedded in switch statement 🔲
**File**: `src/resolver.ts:51-82`
Seven-branch switch mixing URL normalization, env var construction, and AWS credential logic. Adding a new provider type requires editing the resolver directly.

**Fix**: Strategy pattern — one handler per provider type (`BedrockHandler`, `AnthropicHandler`, `ProxyHandler`) each implementing `resolveEnv(provider): Record<string, string>`.

### AWS-1: AWS region not validated against allowed list ✅
**File**: `src/resolver.ts` — `resolvePreset()` now throws with a descriptive message if `awsRegion` is not in `AWS_REGIONS`.

### AWS-2: AWS CLI errors silently swallowed in model discovery ✅
**File**: `src/panel.ts` — each inner catch captures the first error line into `fetchErrors[]`; final error message includes per-command detail plus three-point checklist.

### AWS-3: Hardcoded model list goes stale without republishing ✅
**File**: `src/panel.ts` — `_fetchBedrockModels()` now checks `~/.claude/bedrock-model-cache-<profile>-<region>.json` (1-hour TTL) before calling AWS CLI. Successful fetches are written to cache.

### TEST-1: Zero test coverage 🔲
No test files, no test framework, no CI.
Highest-risk untested code:
- `src/importExport.ts` — credential scrubbing regexes
- `src/resolver.ts` — scope resolution and file writes
- `src/profiles.ts` — store migration logic

**Fix**: Add Jest + ts-jest, write ~25 unit tests for the three modules above, add `.github/workflows/test.yml`.

### IMPORT-1: Merge import does not validate referential integrity 🔲
**File**: `src/importExport.ts:193-195`
After merging, orphaned `providerId` / `mcpGroupIds` / `directoryGroupIds` references are silently written to the store.

**Fix**: After the merge loop, validate all preset references point to existing items. Throw or warn before writing.

### UX-2: New user onboarding gap in scope cards 🔲
**File**: `src/webview/layout.ts:49-56`
New users see a "Preset:" dropdown with no presets and no call-to-action.

**Fix**: When preset list is empty, render inline: "No presets yet — create one below."

### UX-3: Newly created provider does not appear in parent preset drawer 🔲
**File**: `media/webview.js:935-938`
After saving a provider from a nested drawer, the parent preset drawer's provider dropdown is stale.

**Fix**: On drawer close, if parent drawer is still in the stack, rebuild its dropdown HTML without closing it.

### UX-4: Global editing state is fragmented and uses magic numbers 🔲
**File**: `media/webview.js:19-33`
11 module-level variables; `editingMcpServerIndex = -1` means "new".

**Fix**:
```javascript
let editingContext = { type: null, id: null, parentId: null, isNew: false };
```

### UX-5: Inline styles scattered throughout renders 🔲
**Files**: `media/webview.js` (multiple), `src/webview/drawers.ts:54,165`

**Fix**: Extract the 8–10 recurring inline styles to named CSS classes in `styles.ts`.

### IMPORT-2: Draft restoration does not detect externally modified store 🔲
**File**: `src/panel.ts:211-226`

**Fix**: Compare draft timestamp to store file mtime. If store is newer, warn before restoring.

### IMPORT-3: Credential scrubbing placeholders are not context-aware 🔲
**File**: `src/importExport.ts:23`
All scrubbed values become `<REPLACE_ME>`.

**Fix**: Use contextual placeholders: `<REPLACE_ME: anthropicApiKey>`, `<REPLACE_ME: awsProfile>`, etc.

### STATUS-1: Status bar does not distinguish "not configured" from "inherited" ✅
**File**: `src/statusBar.ts` — `$(link)` icon + "Inherited from global" tooltip when inheriting; "Not configured — click to set" tooltip when unset.

### STATUS-2: Quick-switch lacks context and grouping ✅
**File**: `src/statusBar.ts` — placeHolder now shows workspace name; `description` shows `type · providerName` on each preset item.
