How it works

Profiles are reusable provider configurations stored in ~/.claude/provider-profiles.json. Each profile bundles a provider type, credentials, and model selections into a named configuration.
Scopes determine where settings are written. Assign a profile to a scope and the extension writes the correct env vars to that scope's settings.json file.

Scope Precedence (highest wins)

LOCAL .claude/settings.local.json — Your personal override (gitignored)
PROJECT .claude/settings.json — Shared with team via git
GLOBAL ~/.claude/settings.json — Your default for all projects

Sidebar Badges

Each profile in the sidebar shows colored badges indicating which scopes currently use it: G Global, P Project, L Local.
